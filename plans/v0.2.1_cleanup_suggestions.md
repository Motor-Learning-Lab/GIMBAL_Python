## 8. Output Path Standardization Guidelines

### Issue Identified

Some test files save outputs to non-standard locations violating Scheme A:
- `test_ground_truth_init.py` → `results/diagnostics/v0_2_1_divergence/...`
- `debug_model_logp.py` → `results/diagnostics/v0_2_1_divergence/...`
- `test_sanity_posterior_geometry.py` → `plots/` (relative, creates in working dir)
- `test_diagnostics.py` → `results/diagnostics/...` (old TG01-08 pattern)

### Standard Output Path Patterns (Scheme A)

All test outputs must follow these patterns:

**Pattern 1: Test-relative paths (PREFERRED)**
```python
# JSON results
results_path = Path(__file__).parent / "results_<testname>.json"

# Markdown reports  
report_path = Path(__file__).parent / "report_<testname>.md"

# Plots subdirectory
plots_dir = Path(__file__).parent / "plots" / "<group_name>"
```

**Pattern 2: Results subdirectory (for legacy tests only)**
```python
# Only for files already in results/ subdirectory
results_path = Path(__file__).parent / "results" / "results_<testname>.json"
```

**NEVER use:**
- `results/diagnostics/...` (violates Scheme A)
- `Path("plots")` without `__file__.parent` (creates in working directory)
- Hardcoded absolute paths
- Paths traversing to repo root and back

### Files Requiring Fixes

| File | Issue | Fix Required |
|------|-------|-------------|
| `test_ground_truth_init.py` | Uses `results/diagnostics/v0_2_1_divergence/...` | Change to `Path(__file__).parent / "debug_outputs" / "ground_truth_init"` |
| `debug_model_logp.py` | Uses `results/diagnostics/v0_2_1_divergence/debug_logp` | Change to `Path(__file__).parent / "debug_outputs" / "model_logp"` |
| `test_sanity_posterior_geometry.py` | Uses `Path("plots")` (relative to working dir) | Change to `Path(__file__).parent / "plots" / "sanity"` |
| `test_sanity_posterior_geometry.py` | Uses `Path(f"results_{test_name}.json")` | Change to `Path(__file__).parent / f"results_{test_name}.json"` |
| `test_diagnostics.py` | Uses `results/diagnostics/...` | **Archive with TG01-08** (old pattern, not fixing) |
| `report_generator.py` | References `results/diagnostics/...` in text | Update documentation strings to reflect Scheme A |

### Rationale

**Why test-relative paths:**
1. Tests run from any working directory
2. All artifacts stay together in test suite directory
3. Easy to archive entire test suite as unit
4. No confusion about "which results/ directory"

**Why `debug_outputs/` subdirectory:**
- Debug scripts (`debug_model_logp.py`, `test_ground_truth_init.py`) produce diagnostic artifacts, not formal test results
- Keep separate from formal test results/reports
- Still under test suite directory per Scheme A
